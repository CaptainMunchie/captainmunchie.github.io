Interactive Website Design 2/4/2021 - Session 1 2021

Fiesta Journey

MMCC3045 – Interactive Story Telling

To create an online non-linear and interactive web-based story using HTML, CSS, and JQuery skills. It is an interactive story that should be allow the user to influence the flow of the narrative.

Date

2/4/2021 - Session 1 2021

Intro

To create an online non-linear and interactive web-based story using HTML, CSS, and JQuery skills. It is an interactive story that should be allow the user to influence the flow of the narrative.

Project

This interactive story is an adventure, action, comedy, and puzzle game that will be set in the modern world. The story focus on the MC (Main Character) who discovered that his partner has been taken away by some evil antagonist but also discovered later on in his journey that his phone containing all his personal information and other confidential data were taken away with his partner. It is up to the player to help the MC make his decision as to what path should he take, how he would persuade or interact with other characters and lastly how would he end this game.

Highlights

Menu Page

Interactive menu page that has a clickable function represented as buttons. Some buttons will include "start game" button, turning on and off sound button, and go home button. Background music will be added in as well as hover and click sound effects.

Interactive dialogue

Interactive dialogue of two characters communicating with each other. Containing a text bubble, name tag, and drawn character that can be interchanged to express the characters emotion during the communication. This would be done by a variable where it will go through a specific sequence order to follow the storyline. By going through the next text, this will be done by a mouse clickable function, if mouse is clicked, go to next variable. The specific variable will either show or hide the content that is needed for each slide.

Interactive cutscenes

Interactive animated cutscene that would first be hand drawn and animated on adobe anime and converted into a video format such as mp4 and would be automatically played on the cutscene section with sounds included. Other functions such as skipping the video to the next page will be included.

Start, restart and continue game

Each game will start with an instruction that explains how to play the game and click when ready to start playing. If failed, there will be a restart option that goes back to the start unless completed correctly. If completed correctly, it will go to the next page. This will be done and a variable function of 1, 2, 3 and 4. If 1 is true go to 2, if 2 is false go to 3, else go to 4.

Game #1 - home Attack

This game is inspired by Pokémon battle game genre where two character will compete with each other into a strategic battle. How is this done:

By putting it into a variable where the battle follows a consistent pattern and the opponent will attack in a specific sequential order but your character will have other options of attacking the enemy.

Game #2 - Run from enemies

Inspired by the Google Dinosaur T-Rex Game, where the T-Rex is running through the animated moving field while dodging/jumping over objects that is coming towards it. This will be by creating an background image that moves from one side to the other but duplicate itself to the other end creating an illusion of a moving background. The character animated making it look like its moving and jumping on the y-axis while implementing a collision between it and the object saying if object is in contact with the character then game over. A timer will be added saying that if timer reaches 0 and character has not collided with other objects then game is finished.

Game #3 - Survive under water

Enemies will be randomise around the map going in random directions while your character will follow the arrow keys to avoid the enemy being in contact. Collision will be added saying if the character touches the enemy then game over. Timer is also added saying that if timer reaches 0 and character has not collided with other objects then game is finished.

Game #4 - Dodge and Collect

Similar to game #3 the player must also collect the green coins while dodging the black coins at the same time by using the arrow keys to control their character. Collision will be added saying if the character touches the enemy then game over. Timer is also added saying that if timer reaches 0 and character has not collided with other objects then game is finished.

Game #5 - Puzzle game

A puzzle game will be created by breaking an image into 8 pieces and be placed in those 8 tiles where it will be considered full or true. The 9th tile will be empty or false, if the closest image is next to the empty tile, it has the option to move only when the user approved by clicking on it. The image tile will move to the empty tile and will be considered full or true. Once the tiles reaches a specific sequential number the game will be over.

Game #6 - Rocket game

Inspired by flappy bird where the main character is being pushed down by gravity and it is up to the user to tap on the screen to go up for a few second before it goes down and repeat this process while avoiding the obstacle at the same time. To do this, is by making the character to constantly fall down on a y-axis and if the user press "space" key then the character will go up at a certain height for certain duration. Background will be animated to create the illusion of a moving scenery done by moving the first image from point A to point B while repeatedly adding that background next to it. Obstacles will be added for the character to avoid, this is done with collision where if the character touches the object them it will hold true where the game will be over.

Game #7 - Boss fight

A dodge game will be created where moving objects will move randomly, while the user uses the arrow keys to avoid them. By using collision detection so that if the object is in contact with the character then game over. Timer will be added, once the time has reached 0, the boss will appear and move randomly. The user must click on the boss to deal damage within another set of time. This process will be repeated until all the bosses health has reached 0 then game over.

More Information page
